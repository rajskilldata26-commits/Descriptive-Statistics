Q1.What is a null hypothesis (H₀) and why is it important in hypothesis testing?
SOL:-A null hypothesis (H₀) is a statement that assumes no effect, no difference, or no relationship between variables. It represents the default or baseline assumption that any observed result is due to chance or random variation.
Example
If you are testing a new medicine:
1.H₀ (null hypothesis): The medicine has no effect on patients.
2.Alternative hypothesis (H₁): The medicine does have an effect.

Why is the null hypothesis important?
1. Provides a starting point
It gives a clear assumption to test against. Without H₀, you wouldn’t have a benchmark to compare results.

2. Helps in decision-making
Hypothesis testing works by checking whether there is enough evidence to reject H₀.
If evidence is strong → reject H₀
If evidence is weak → fail to reject H₀

3. Supports objective analysis
It prevents bias by forcing you to assume “no effect” until data proves otherwise.

4. Forms the basis of statistical tests
Tests like z-test, t-test, chi-square test all revolve around evaluating H₀.

Question 2: What does the significance level (α) represent in hypothesis testing?
SOL:-The significance level (α) in hypothesis testing is the probability of rejecting the null hypothesis (H₀) when it is actually true. In simple terms, it measures how much risk you are willing to take of making a wrong decision.

What does it represent?
1.α is the threshold for deciding whether a result is statistically significant
2.It defines the maximum acceptable probability of a Type I error

Type I Error
This happens when:
You reject H₀ even though it is true (a “false positive”)
So,
α=P(Type I Error)
Common values of α
1.0.05 (5%) → most commonly used
2.0.01 (1%) → stricter test
3.0.10 (10%) → more lenient test
How it works in testing
If p-value ≤ α → Reject H₀ (result is statistically significant)
If p-value > α → Fail to reject H₀
Simple example
Suppose:
α = 0.05
This means you accept a 5% chance of wrongly concluding that something has an effect when it actually doesn’t

Question 3: Differentiate between Type I and Type II errors
SOL:-In hypothesis testing, Type I and Type II errors are two kinds of mistakes you can make when deciding about the null hypothesis (H₀).
🔹 Type I Error (False Positive)
 Definition: Rejecting the null hypothesis (H₀) when it is actually true
 Meaning: You conclude there is an effect when there is none
 Probability: Equal to the significance level α
Example
A test says a medicine works, but in reality, it doesn’t.

🔹 Type II Error (False Negative)
 Definition: Failing to reject the null hypothesis (H₀) when it is false
 Meaning: You conclude there is no effect when there actually is one
 Probability: Denoted by β
Example
A test says a medicine does not work, but it actually does.

| Aspect      | Type I Error          | Type II Error                 |
| ----------- | --------------------- | ----------------------------- |
| Decision    | Reject H₀ incorrectly | Fail to reject H₀ incorrectly |
| Reality     | H₀ is true            | H₀ is false                   |
| Also called | False Positive        | False Negative                |
| Probability | α                     | β                             |

Question 4: Explain the difference between a one-tailed and two-tailed test. Give an example of each.
SOL:-The difference between a one-tailed test and a two-tailed test lies in what kind of effect you are testing for and where the rejection region is located

🔹 One-Tailed Test
A one-tailed test checks for an effect in only one direction (either greater than or less than).
Types:
 Right-tailed test: Tests if a value is greater than a certain point
 Left-tailed test: Tests if a value is less than a certain point
Example:
H₀: The average score ≤ 50
H₁: The average score > 50

🔹 Two-Tailed Test
A two-tailed test checks for an effect in both directions (either greater than or less than).
Example:
H₀: The average score = 50
H₁: The average score ≠ 50
Here, you care if the score is either higher or lower than 50.

| Aspect           | One-Tailed Test         | Two-Tailed Test             |
| ---------------- | ----------------------- | --------------------------- |
| Direction        | One direction only      | Both directions             |
| Alternative (H₁) | > or <                  | ≠                           |
| Rejection region | One side (one tail)     | Both sides (two tails)      |
| Use case         | When direction is known | When any difference matters |

At α = 0.05, test the claim.
SOL:-Step 1: Set up hypotheses

Step 2: Given data
Sample mean

Substitute values:

Step 4: Critical value
Degrees of freedom: df=n−1=8
For α=0.05 (two-tailed), critical value ≈ ±2.306

Step 5: Decision
Calculated t=2
Critical t=±2.306

Since:
 ∣2∣<2.306
 Fail to reject H₀

Step 6: Conclusion
There is not enough evidence at the 5% significance level to reject the company’s claim.

Question 6: When should you use a Z-test instead of a t-test.
SOL:-You should use a Z-test instead of a t-test when certain conditions about the data are met—mainly related to sample size and knowledge of population variability.

🔹 Use a Z-test when:
1. Population standard deviation (σ) is known
 This is the most important condition
 In real life, this is rare but assumed in some problems
2. Large sample size
Typically n ≥ 30
 With large samples, the sampling distribution becomes approximately normal (Central Limit Theorem)
3. Data is approximately normally distributed
 Especially important for smaller samples (though Z-test is mainly for large samples)
🔹 Use a t-test when:
 Population standard deviation (σ) is unknown
 Sample size is small (n < 30)
 You use sample standard deviation (s) instead of σ

| Aspect                   | Z-test                  | t-test          |
| ------------------------ | ----------------------- | --------------- |
| Standard deviation       | Known (σ)               | Unknown (use s) |
| Sample size              | Large (≥ 30)            | Small (< 30)    |
| Distribution used        | Normal (Z distribution) | t-distribution  |
| Accuracy (small samples) | Less accurate           | More accurate   |

Question 8: A company wants to test if product preference is independent of gender.
Gender  ProductA  ProductB  Total
MALE     30       20        50
FEMALE   10       40        50
Total    40       60        100
At α = 0.05, test independence
SOL:-This is a Chi-square test of independence.
🔹 Step 1: Hypotheses
H₀: Product preference is independent of gender
H₁: Product preference is dependent on gender
🔹 Step 2: Observed frequencies (O)
| Gender | A  | B  | Total |
| ------ | -- | -- | ----- |
| Male   | 30 | 20 | 50    |
| Female | 10 | 40 | 50    |
| Total  | 40 | 60 | 100   |
🔹 Step 3: Expected frequencies (E)
Formula:E=(Row Total × Column Total)/Grand Total
Male–A: 20
Male–B:=30
Female–A: =20
Female–B:=30
🔹 Step 4: Chi-square statistic
χ^2=∑(O−E)^2/E	
χ^2==5+3.33+5+3.33=16.66

🔹 Step 5: Critical value
Degrees of freedom:
df=(r−1)(c−1)=(2−1)(2−1)=1
At α = 0.05 → Critical χ² ≈ 3.84

🔹 Step 6: Decision
Calculated χ² = 16.66
Critical χ² = 3.84
Since:
16.66>3.84
Reject H₀

🔹 Step 7: Conclusion
There is strong evidence at the 5% significance level that:
Product preference is not independent of gender
Gender and product choice are associated (dependent).
